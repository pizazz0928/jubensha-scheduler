# 归来剧场 · 实时排班系统

面向盒装剧本杀门店的当日实时调度 MVP，重点解决临时开本、快速换 DM、DM 玩家补位、房间/人员冲突与完整操作留痕。

## 已实现

- 今日调度大屏：场次统计、风险提醒、时间轴、DM 实时池
- 临时开本：填写剧本、人数、时间和房间，采用推荐方案一键创建
- 快速换 DM：显示推荐分、推荐原因和连锁影响
- DM 玩家补位：人数检查、候选排序、补位后状态联动
- DM 状态：空闲、准备、主持、补位和休息的快速切换
- 剧本、房间、调度记录与工作量统计页面
- D1 持久化操作日志与 Drizzle 数据迁移
- DM、剧本与房间完整资料库：新增、编辑、停用、恢复和状态操作
- DM 与剧本熟练度关联，最低熟练度真实参与推荐
- 自定义房间容量、类型和清理状态真实参与临时开本推荐
- 手机浏览器适配

## 本地启动

需要 Node.js 22.13 或更高版本。

```bash
pnpm install
pnpm dev
```

浏览器打开开发服务器显示的本地地址。

## 数据库

站点使用 Cloudflare D1，逻辑绑定名为 `DB`。数据表定义在 `db/schema.ts`，已生成迁移文件。

修改表结构后重新生成迁移：

```bash
pnpm db:generate
```

调度 API 位于 `/api/dispatch`。首次调用会安全创建所需表，关键调度动作同时写入操作日志。

## 测试与构建

```bash
pnpm test
pnpm build
```

规则测试覆盖 DM 时间冲突、主持/补位身份冲突、剧本资格、房间冲突、补位许可与人数上限。

## 演示方式

1. 点击右下角“临时开本”，选择剧本并采用推荐方案。
2. 在缺 DM 场次点击“安排 DM”，比较三位候选人的评分和影响。
3. 在缺玩家场次点击“添加补位”，确认后玩家数和 DM 状态会联动更新。
4. 在“DM 状态”新增人员并维护主持、补位、到店和可用时间。
5. 在“剧本管理”设置人数、时长、补位规则，并为 DM 配置 1–5 级熟练度。
6. 在“房间管理”维护容量、房态、适配类型和清理状态。
7. 再次临时开本时，系统会立即使用这些自定义资料进行推荐。

演示数据包含 8 个今日场次、缺 DM、缺玩家、DM 补位、房间冲突，以及发生过三次交接的场次。
